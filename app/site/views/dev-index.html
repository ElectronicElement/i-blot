Here you’ll find documentation on all the different ways to create templates on Blot. A working knowledge of HTML and CSS is neccessary to develop templates on Blot. Blot’s templates are dedicated to the public domain so you can use, modify and share them freely.

Developing your own template

Blot's template system offers you, the developer, complete control over your blog's routes. This means you can control every line of HTML, CSS and JavaScript served to your readers. Everything from robots.txt to your blog's CSS file.

Getting started

You can create a new template on the design page of your blog’s dashboard. You can make changes to a template using the online editor, or you can sync the template’s source files to your blog’s folder and edit them locally.

Template files

Blot’s templates will be familiar to those who have worked with other blogging platforms. They are written in Mustache and consist of tags which look like {{this}}. These tags are replaced with the data generated by Blot when the template is rendered.

Routing

When a request from one of your readers arrives at your blog, Blot follows these steps to work out what to send in response.

  1. Check if there is a template whose route matches the request’s path. Render the template if it matches.
  2. Check if there is a file which matches the request’s path in your blog’s folder. Send the file as a response if it matches.
  3. Check your site's list of redirects.
  4. Render your blog’s error page.
