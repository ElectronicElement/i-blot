Blot is a blogging platform. It creates a special folder in your Dropbox and publishes files you put inside automatically. There’s no publishing interface, just files and a folder. This means you can use your favorite text‐editor to write your blog posts.

Blog posts

Any of these file types become blog posts automatically when you put them in your folder:

Text and Markdown  .txt  .md
Word Documents  .docx
Images  .png  .jpeg  .gif
Bookmarks  .webloc  .url
HTML  .html

You can add links, headings, images, code snippets and much more to your blog posts. Read the formatting guide to learn more.

Pages and drafts

Files in a folder called Pages are added to your blog’s menu instead of the list of blog posts.

Files in a folder called Drafts are not published. Instead, Blot creates a preview file which you can open in your browser to see how the post will appear on your blog.

Every file is public

The files in your blog’s folder are public and can be accessed by your readers. Use this feature to distribute files and embed media in your blog posts. Read more about serving static files.

Ignoring files

Blot skips files and folders whose name starts with an underscore. Ignored files do not become blog posts but they are public.

Metadata

The metadata for each blog post is generated automatically from the file’s contents, name and location. This metadata includes a permalink and publish date. You can override this metadata in the first few lines of a file:

Date: {{#date}}MMMM Do, YYYY{{/date}}
Permalink: /metadata

Metadata must start on the file’s first line and must be separated from the rest of the post by at least one blank line.

View all the metadata which you can specify at the start of a file. You can also define your own metadata for use in your blog's template.

Dates

Blot uses the file’s creation date as its publish date by default. You can specify a different date in the file’s metadata like this:

Date: {{#date}}MMMM Do, YYYY{{/date}}

Blot supports a variety of date formats. The following all produce a post with the same date:

{{#date}}MM/DD/YYYY{{/date}}    {{#date}}MMMM D, YYYY{{/date}}
{{#date}}M.D.YYYY{{/date}}     {{#date}}YYYY-MM-DD 00:00{{/date}}

You can also use the file’s name or its path:

{{#date}}YYYY-MM-DD-00:00[-Post.txt]{{/date}}    {{#date}}YYYY/MM/DD/[Post.txt]{{/date}} 
{{#date}}YYYY/M.D.[Post.txt]{{/date}}           {{#date}}YYYY/M.D/[Post.txt]{{/date}}

You can schedule a blog post for future publication. If you specify a future publish date, Blot will wait until then to publish it. If you prefer to write your dates in a different order, please update your date preferences on your blog’s dashboard.

Tags

You can group posts which cover similar subjects using tags. You can tag a post in the metadata at the top of a file:

Tags: Getting started, Documentation

Tags are case-insensitive and may contain whitespace. Blot uses the most recent case when displaying a tag on your blog.

You can use brackets in a file or folder’s name to add additional tags to a blog post. For example, Ada.txt will have the tag 'Books' and Thief.txt will have the tag 'Films':


[Books]
  Ada.txt
[Films]
  Thief.txt

Permalinks and URLs

Blot generates a permalink for each post. By default, Blot uses a URL-friendly version of the post’s title. You can change the format of you blog’s permalinks on the dashboard or you can also specify a permalink in the file’s metadata:

Permalink: /how-to-change-a-url

If you change a post’s permalink, Blot will redirect requests to its former URL automatically.
