Blot is a blogging platform. It creates a folder inside your Dropbox and publishes files you put inside. These files become blog posts automatically:

  • Text .txt, .md
  • Images .png, .jpeg, .gif
  • Word documents .docx
  • Bookmarks .webloc
  • HTML .html

Public files

All the files in Blot’s folder are public. For example, if you put orange.zip in your folder, your readers can download it at your-website.com/orange.zip.

Skipping files and folders

Blot will ignore files and folders whose name starts with an underscore. For example, Banana.jpg will become a blog post but _Apple.jpg will not.

Pages

Files in a folder called Pages are added to your blog’s menu and do not become blog posts. You can also create a page using the file’s metadata.

Drafts

Files inside a folder called ‘Drafts’ are not published to your site. Instead, Blot creates a preview for that file. Open the preview file to see how the post will appear on your site. You can also create a draft using a file’s metadata.

Formatting blog posts

Blot supports Markdown. It can also parse Mathematical statements and code snippets. View the full formatting reference.

Metadata

Blot generates a date, permalink, thumbnail and a list of tags for each blog post automatically. You can override them:

Date: February 28th, 1973
Tags: Literature, Nazis

# Gravity's Rainbow

A screaming comes across the sky.

Metadata must start on the first line and be separated from the rest of the post by a blank line. Read the list of all metadata.

Permalinks and URLs

Blot generates a permalink for each post automatically. You can specify the format of your site’s permalinks on the dashboard. You can also specify a permalink in the metadata at the top of a file:

Permalink: /heautontimorumenos

Enter CHREMES, and MENEDEMUS with a spade...

Dates

You can specify a publish date in the file’s metadata:

Date: 23/06/2012

# You and the Atomic Bomb

Considering how likely we all are to be blown...

Blot supports a variety of ways that people write dates. Each of the following formats produces a blog post with the same publish date:

23/06/2012    23.6.12    23-6-2012    2012-6-23 00:00

You can also use the file’s path to specify a publish date. These will all have a publish date of midnight on February 23rd, 2016:

/2016/2/23/post.txt    /2016/2.23.post.txt   2016-02-23-00:00-post.txt

If you write the numbers in your dates in different order, please tell Blot on the settings page.

If you specify a date in the future, Blot will wait until then to publish it.

If you don’t specify a date, the file’s creation date is considered its publish date.

And don’t forget, you can hide the dates on your site if you prefer.

Tags

Tags are used to associate posts by subject. Many of Blot’s templates include pages to display posts matching a given tag.

You can specify tags in the metadata at the top of a file:

Tags: Littérature, Book reviews

Tags can contain whitespace and should be separated by a comma.

Tags are case-insensitive. For example, the tags Proust, PROUST and proust are equivalent. However, Blot displays the case of the most recently used tag on your blog so it’s worth capitalizing tags consistently.

Using files and folders to tag posts

You can use folders to tag posts. For example, files in a folder called [Literature] will become blog posts tagged ‘Literature’. You can nest tag folders:

[Littérature] / [Book reviews] / Lamiel.txt
