Blog posts

All of these files become blog posts automatically when you put them somewhere inside your blog’s folder:

  • Text and markdown Use the full range of markdown features in these files, including footnotes.
  • Word documents Most of the typographic formatting is stripped.
  • Bookmarks Drag URLs from your browser's address bar to create these files. Blot turns them into a simple link.
  • Images Drag URLs from your browser's address bar to create these files. Blot turns them into a link back to the original site.
  • HTML Use HTML fragments. Blot will leave them as is.

Any changes you make are applied automatically to your site. See Drafts

Organize your folder as you like. You can organize the files in your folder to suit you and your blog. Create subfolders if you like, or don't. You can always re-organize your folder later.

Add formatting to your posts. Blot supports a variety of standard tools to add formatting to your blog posts. You can use Markdown to add links, headings, lists and much more. Blot also renders mathematics set in \(\TeX\) and provides syntax highlighting for code snippets.

Read the formatting guide 
# Formatting

You can use *Markdown*, a lightweight markup language for plain text files.

Use it for links, headings, lists and much more.

Drafts

Files in a folder called ‘Drafts’ are not published. Instead, Blot creates a preview file which updates as you make edits.

Pages

Files in a folder called ‘Pages’ become pages instead of blog posts. They are added to your blog’s menu.

Metadata

Each blog post’s permalink and date is generated automatically from the file’s contents and location. You can override this metadata at the start of a file.

How to change a post’s date How to add tags to a post View the metadata reference 
Date: {{#date}}MMMM Do, YYYY{{/date}}
Permalink: /metadata

Metadata must start on the file’s first line and must be separated from the rest of the post by at least one blank line. You must leave a space after the colon.

Public files

Every file in your blog’s folder is public. Use this feature to distribute files to your readers and embed media in blog posts.
