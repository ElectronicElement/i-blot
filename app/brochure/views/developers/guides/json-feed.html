JSON feed

The JSON feed format is a pragmatic syndication format, like RSS and Atom, but with one big difference: itâ€™s JSON instead of XML.

The template

Create a new view and paste in the following template. You can use the JSON feed validator to verify your feed is correctly formed.

Warning: If you have an existing view called feed.rss or feed.xml, please name your JSON feed something other than feed.json. There's an obscure and silly bug which is triggered when two template views share the same name, even with different extensions. I know this is bad and I will fix it soon.

{ 
  "version": "https://jsonfeed.org/version/1",
  "title": "\{{#encodeJSON}}\{{{title}}}\{{/encodeJSON}}",
  "description": "\{{#encodeJSON}}Feed for \{{{title}}}\{{/encodeJSON}}",
  "home_page_url": "\{{{blogURL}}}",
  "feed_url": "\{{{blogURL}}}/jsonfeed.json",
  "items": [
    \{{#recentEntries}}
    { 
      "id": "\{{{absoluteURL}}}",
      "title": "\{{#encodeJSON}}\{{{title}}}\{{/encodeJSON}}",
      \{{#summary}}
      "summary": "\{{#encodeJSON}}\{{{summary}}}\{{/encodeJSON}}",
      \{{/summary}}
      \{{#thumbnail.large}}
      "image": "\{{{blogURL}}}\{{{url}}}",
      \{{/thumbnail.large}}
      "content_html": "\{{#encodeJSON}}\{{#absoluteURLs}}\{{{body}}}\{{/absoluteURLs}}\{{/encodeJSON}}",
      "date_published": "\{{#formatDate}}YYYY-MM-DDTHH:mm:ssZ\{{/formatDate}}",
      "url": "\{{{absoluteURL}}}"
    }\{{^last}},\{{/last}}
    \{{/recentEntries}}
  ]
}
